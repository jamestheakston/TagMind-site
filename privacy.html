Privacy Policy

Last updated: July 2026

1. Stateless Architecture

TagMind is engineered as a stateless automation utility. We do not run persistent databases, user logging clusters, or cache systems. When a GitHub repository webhook delivers data to our serverless edge handlers, the raw event parameters are handled explicitly within ephemeral execution memory and instantly destroyed once processing wraps up.

2. Data Processing & Isolation

We process repository event details, issues, and pull request information solely to identify text markers and map relevant organizational tags. Your files, programmatic code contents, and confidential repository configurations are never viewed, extracted, compiled, or transmitted to external nodes.

3. Least Privilege Authorization

The application functions strictly under localized GitHub parameters requesting metadata read and write rights on issue structures and pull requests. TagMind explicitly holds zero administrative authority over authorization schemes, structural profiles, system keys, or primary file architectures.

4. Third-Party Integrations

No consumer usage patterns, text entries, tracking configurations, or client statistics are transferred to outside services, analytics firms, or marketing networks. All communication pipelines stay encapsulated purely between your environment, our serverless execution edge, and GitHub's core secure infrastructure portals.

For explicit clarifying points regarding this transparency setup or structural pipeline rules, connect with technical support directly through our communications desk at jamesttheakston2@gmail.com.
